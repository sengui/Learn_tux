Discovering a Tuxedo domain

Use the discovery page to discover EM targets inside a Tuxedo domain. After the first discovery succeeds, EM will automatically update targets in the Tuxedo domain every day.

Following parameters should be supplied for Tuxedo target discovery:

  • Hostname: Mandatory parameter. Specifies the host where the master machine of Tuxedo domain is running.

  • Port: Mandatory parameter. Specifies the port listened by RMI connector of JMX agent embedded in "tlisten" process on the master machine of Tuxedo domain.

  • Application Password: Optional parameter. Specifies the Tuxedo application password with which the EM agent connects the Tuxedo domain. You need to input this parameter if the SECURITY parameter value of the Tuxedo domain is one of following: APP_PW, USER_AUTH, ACL or MANDATORY_ACL; otherwise, leave it empty.

  • User name: Optional parameter. Specifies the Tuxedo user name with which the EM agent connects the Tuxedo domain. You need to input this parameter if the SECURITY parameter value of the Tuxedo domain is one of following: USER_AUTH, ACL or MANDATORY_ACL; otherwise, leave it empty.

  • User Password: Optional parameter. Specifies the Tuxedo user password with which the EM agent connects the Tuxedo domain. You need to input this parameter if the SECURITY parameter value of the Tuxedo domain is one of following: USER_AUTH, ACL or MANDATORY_ACL; otherwise, leave it empty.

  • Use SSL: Optinal.

  • With Tuxedo Authentication: If this box is not checked, all Tuxedo security relative information are not required, and only tlisten and Tuxedo Home target will be discovered.

  • Monitoring Agent: Mandatory option, by default the one on the same physical machine along with the tlisten will be used
