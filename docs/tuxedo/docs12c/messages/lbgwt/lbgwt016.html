LIBGWT Messages 1600-1699

 

1601

INFO: Only Triple-DES is supported for session authentication by ldom!

Description

The environmental variable GWT_SNP_3DES is set to "YES" for local GWTDOMAIN gateway ldom. The local GWTDOMAIN gateway ldom will only accept or establish connection with remote GWTDOMAIN gateway that supports using Triple-DES for session authentication. The default value for "GWT_SNP_3DES" is "NO".

Action

No action is required, this is informational. If this is not what you want then change the environmental variable value to NO.

1602

ERROR: Triple-DES is required by ldom but remote domain <rdom> does not support it, outbound connection request rejected!

Description

The local GWTDOMAIN gateway ldom requires Triple-DES for session authentication, but the remote GWTDOMAIN gateway rdom is running on an older version and does not support it. This happens when SECURITY is configured with either APP_PW or DM_PW, and local GWTDOMAIN gateway ldom is the connection initiator.

Action

Either upgrade the remote GWTDOMAIN Tuxedo installation to a release or patch level that supports Triple-DES, or change the configuration to NONE, or change the environmental variable "GWT_SNP_3DES" to "NO".

1603

ERROR: Triple-DES is required by ldom but remote domain <rdom> does not support it, inbound connection request rejected!

Description

The local GWTDOMAIN gateway ldom requires Triple-DES for session authentication, but the remote GWTDOMAIN gateway rdom is running on an older version and does not support it. This happens when SECURITY is configured with either APP_PW or DM_PW, and local GWTDOMAIN gateway ldom is the connection responder.

Action

Either upgrade the remote GWTDOMAIN Tuxedo installation to a release or patch level that supports Triple-DES, or change the SECURITY configuration to NONE, or change the environmental variable "GWT_SNP_3DES" to "NO".

1604

WARN: TDomain Session Negotiation timed out by ldomid, SNP state <snp_state>; close connection to remote domain<rdomid, rdom_naddr>!

Description

It took longer than expected to complete Session Negotiation for connection between local GWTDOMAIN gateway ldomid and remote GWTDOMAIN rdomid. The host IP address of the remote GWTDOMAIN gateway is rdom_naddr. The last SNP state for that connection is snp_state.

Action

The administrators have to checkout possible connection flooding attack. If not being attacked then check out why the communication between the two gateway is slow, and correct the problem accordingly. Or may be need to make the SNP timeout value larger or disable it. By default there is no SNP timeout, it has to be enabled through environmental "GWT_SNP_TIMEOUT".

1605

WARN: TDomain Session Negotiation State timed out by ldomid, SNP state <snp_state>; close connection to remote domain<rdomid, rdom_naddr>!

Description

It took longer than expected to transition from current Session Negotiation state to the next Session Negotiation state for connection between local GWTDOMAIN gateway ldomid and remote GWTDOMAIN rdomid. The host IP address of the remote GWTDOMAIN gateway is rdom_naddr. The last SNP state for that connection is snp_state. In the case of environment variable GWT_VALIDATE_HOST is not set to "YES", and GWT_ALLOW_FROM_ANY_HOST is set to NO, GWTDOMAIN will accept any incoming connections. This will open a door of DOS attack if GWTDOMAIN doesn't run in a secure environment.

Action

The administrators have to checkout possible connection flooding attack. If not being attacked then check out why the communication between the two gateway is slow, and correct the problem accordingly. Or may be need to make the SNS timeout value larger or disable it. By default there is no SNS timeout, it has to be enabled through environmental "GWT_SNS_TIMEOUT".

1612

INFO: Validate host feature is enabled by ldom.

Description

This feature validates the host address for the incoming connection request. It is enabled by set an environmental variable "GW_VALIDATE_HOST" to "YES". By default this feature is disabled. When it is enabled GWTDOMAIN gateway will check the host IP address to see whether it is from one of the host configured.

Action

No action is required.

1613

INFO: Session Negotiation Protocol timeout set to sec seconds is being enforced by ldom.

Description

The environmental variable GWT_SNP_TIMEOUT is set to a value between 0 and MAXINT. By default the SNP timeout is disabled, if enabled then all the session negotiation must be complete before the timer expires.

Action

No action is required.

1614

Description

Action

1615

INFO: Session Negotiation State timeout set to sec seconds is being enforced by ldom.

Description

The environmental variable GWT_SNS_TIMEOUT is set to a value between 0 and MAXINT. By default the SNS timeout is disabled, if enabled then session negotiation internal must transition to one of the possible next states before the timer expires.

Action

No action is required.

1618

INFO: Invalid host access logging is explicitly turned off by ldom!

Description

Action

No action is required.

ERROR: MAC is MANDATORY but remote domain <rdomid> doesn't support this feature

Description

If MAC is set to MANDATORY in local domain, and the remote domain does not support/deploy the MAC feature, the session cannot be established.

Action

Contact remote domain administrator to re-configure MAC feature support. It may be necessary to install a later Tuxedo rolling patch version.

1682

ERROR: MAC is MANDATORY but neither LLE nor SECURITY is supported for connection of (<LDOM>, <RDOM>)

Description

If MAC is set to MANDATORY in local/remote domain, but LLE or domain SECURITY is not supported for the connection between local domain and remote domain, the session cannot be established.

Action

Check whether LLE and/or domain SECURITY is deployed in local/remote domain. If already deployed in local domain, then contact remote domain administrator to re-configure LLE and/or domain SECURITY support. Otherwise, don't configure MAC as MANDATORY in local/remote domain.

Action
